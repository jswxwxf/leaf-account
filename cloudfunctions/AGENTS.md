# 云函数 (后端) 中的 AI 代理

本项目利用 AI 代理来协助开发和维护云函数后端。 这些代理遵循项目指南、编码规范和最佳实践，负责代码生成、错误修复，并根据项目指南和最佳实践提出改进建议，特别是对于后端。

## 核心技术

*   **平台**: CloudBase (腾讯云)
*   **语言**: Node.js
*   **数据模型操作**: CloudBase Models SDK
*   **云函数运行时**: Node.js 18.15
*   **规则**: 遵循 `cloudbase-rules.md` 和 `coding_standards.md` 中定义的规则

## AI 代理角色

*   **函数生成**:
    *   根据需求生成云函数代码，例如数据处理、API 接口等。
    *   使用 `createFunction` 工具创建云函数，使用 `updateFunctionCode` 工具更新云函数代码。
    *   确保代码符合 `coding_standards.md` 中定义的规范。
*   **数据访问**:
    *   使用 CloudBase Models SDK 操作数据库，遵循 `database.mdc` 中定义的数据库操作规则和注意事项。
    *   根据业务需求，选择合适的数据库权限，确保数据安全。
    *   通过云函数实现跨数据库集合的操作。
*   **安全性**:
    *   建议安全性最佳实践，例如对用户输入进行验证、防止 SQL 注入等。
    *   识别潜在漏洞，并提出修复建议。
*   **错误修复**: 分析云函数日志，并根据 CloudBase 官方文档和最佳实践，提出修复错误的解决方案。
*   **性能优化**: 关注云函数的性能，包括执行时间、内存使用等，并提出优化建议。
*   **文档**: 为云函数和 API 生成清晰、易懂的文档，并符合 JSDoc 风格。

## 专业领域规则文件

AI 代理在云函数开发场景中会参考以下规则文件：

*   `rules/cloudbase-platform.mdc`: 描述 CloudBase 平台的核心知识，包含云开发环境和认证、云函数、数据库、存储等服务、数据模型和权限管理等。
*   `rules/database.mdc`: 描述云开发 CloudBase 数据库操作的专业规则，包含 CloudBase 数据库操作注意事项、数据库权限管理、数据更新和错误处理等。
*   `rules/workflows.mdc`: 描述开发工作流程，包含部署流程、素材下载和知识库查询、文档和配置文件生成规则、MCP 接口调用规范等。
